## `Access Specifier Indent` <Badge text="Detected" variant="success"  size="medium" aria-label="error detected"/>
Access specifiers like `public`, `protected`, and `private` should be indented by 1-space.
```cpp title="myclass.h"
class MyClass {
 public:
  void Print() const;
};
```
[Reference to `cpplint` error](./cpplint-reference#access-specifier-indent)

## `Constant` <Badge text="Not Detected" variant="danger"  size="medium" aria-label="error not detected"/>
Constants should be formatted with a lowercase `k` followed by `CamelCasing` the rest of the name.
```cpp
const double kPie = 3.14;
```
Error not detected by `cpplint`

## `Control Structures` <Badge text="Detected" variant="success"  size="medium" aria-label="error detected"/>

- Blank space between the statement type and the parenthesis.

- No blank spaces after the opening parenthesis and before the closing parenthesis.

- Single space between the parenthesis and the opening brace.

- Opening brace should be on the same line as the statement.

- No blank lines between the braces and the code block.

## `Copyright` <Badge text="Detected" variant="success"  size="medium" aria-label="error detected"/>
The first line of each file should be authored with a comment in the following format `// Copyright <year> <name>`.
```cpp
// Copyright 2024 CSCE240
```
[Reference to `cpplint` error](./cpplint-reference#copyright)

## `Function` <Badge text="Mostly Detected" variant="success"  size="medium" aria-label="error detected"/>
- The name of the function should start with a capitalized letter and casing should follow CamelCase rules. This is not caught by `cpplint`.

<Aside>The `main` function is an exception and should **ALWAYS** be written as `main()`.</Aside>

- Parenthesis should be next to the function's name.

- There should **NOT** be a space after the opening parenthesis, or before the ending parenthesis.

- There should be a single space between the opening parenthesis, and the opening brace.

- Opening braces should **ALWAYS** be in the same line as the associated statement. If the function implementation is only 
one line small line, you can place the closing bracket on the same line.

## `Variable` <Badge text="Not Detected" variant="danger"  size="medium" aria-label="error not detected"/>
Variables should be named using `snake_case`.
```cpp

```
